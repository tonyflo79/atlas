# Proposal vs. Mutation — Atlas API Surface Audit

**The invariant Atlas's safety story rests on:**

> No graph-typed state mutation happens automatically as part of Ripple
> propagation. Reading the graph and *proposing* changes is one operation.
> *Applying* a change to the graph is a separate operation that goes
> through `adjudication.resolve()` (after a human or rule decides) or the
> AGM `revise()` / `contract()` operators directly. There is no path
> from `ripple.analyze_impact()` or `ripple.reassess()` that ends in a
> mutated typed graph.

If a method appears in this document on the wrong side of the line, that's
a bug in the audit — file an issue. If a method appears on the wrong side
in the *code*, that's a bug in the safety guarantee — file a P0 issue.

This document is regenerated by hand on every release; the next refresh is
due whenever a new MCP tool, AGM operator, or trust-store method ships.

Spec: docs/LAUNCH_BACKLOG.md → P1 "Proposal-vs-mutation explicit in API surface".

---

## How to read this

Every public method in `atlas_core/ripple/`, `atlas_core/revision/`,
`atlas_core/trust/`, and `atlas_core/api/` is classified into exactly
one of three buckets:

- **READ-ONLY** — issues only `MATCH` Cypher (no `CREATE`, `MERGE`, `SET`,
  `DELETE`), and writes nothing to SQLite. Pure observation. Safe to
  call at any time, in any order, by any caller. Cannot violate AGM
  postulates by definition.
- **PROPOSAL** — produces a structured suggestion (a
  `ReassessmentProposal`, `RoutingDecision`, or `ContradictionReport`).
  May write a markdown file to the Obsidian adjudication queue (a
  *queue* side-effect, not a graph mutation). Does not modify the
  typed graph, the AGM closure, or the SHA-256 ledger chain. A
  proposal that's never resolved leaves Atlas's state unchanged.
- **MUTATION** — modifies the typed graph (adds nodes/edges, sets
  properties, creates `:AtlasRevision` chains), or appends to the
  hash-chained ledger, or changes quarantine candidate status.
  Calling this is committing to a state change. Every mutation must
  either flow from a human-resolved adjudication entry, an AGM
  operator invoked deliberately, or the trust-promotion path —
  never from a Ripple cascade alone.

A handful of methods touch *non-graph* persistent state — quarantine
SQLite, ledger SQLite, sharing-policy SQLite. Those are called out as
**MUTATION (non-graph)** so the invariant about the *typed graph*
stays sharp.

## The audit

### `atlas_core/ripple/` — the cascade

| Method | Class | What it does |
|---|---|---|
| `analyze_impact(driver, kref, ...)` | **READ-ONLY** | Recursive Cypher walk of `DEPENDS_ON` from `kref`. Returns `AnalyzeImpactResult` with impacted-node list + cycle report. Zero writes. |
| `reassess_dependent(driver, dep_kref, change, ...)` | **READ-ONLY** | Reads dep node's confidence + `DEPENDS_ON` strength + temporal-decay metadata. Returns one `ReassessmentProposal`. Zero writes. |
| `reassess_cascade(driver, impacted, change)` | **READ-ONLY** | Calls `reassess_dependent` per impacted node. Returns `list[ReassessmentProposal]`. Zero writes. |
| `detect_contradictions(driver, proposals)` | **READ-ONLY** | Type-aware contradiction detector. Reads node labels + neighbours. Returns `list[ContradictionReport]`. Zero writes. |
| `route_proposal(driver, proposal, contradictions)` | **PROPOSAL** | Computes `RoutingDecision` (auto-apply / strategic-review / core-protected). Writes a markdown adjudication-queue entry to disk if the routing is strategic-review. **No graph mutation.** |
| `route_all(driver, proposals, contradictions)` | **PROPOSAL** | Bulk wrapper around `route_proposal`. Same guarantees. |
| `write_adjudication_entry(...)` | **PROPOSAL** | Pure markdown writer. Filesystem queue, not graph. |
| `write_strategic_entries(...)` | **PROPOSAL** | Bulk markdown writer. Same. |
| `RippleEngine.propagate(upstream_kref, ...)` | **PROPOSAL** | Orchestrates `analyze_impact` → `reassess_cascade` → `detect_contradictions` → `route_all`. Returns `CascadeResult`. **The whole cascade ends in proposals + queue entries; the typed graph is unchanged.** |
| `resolve_adjudication(proposal_id, decision, driver, ledger, ...)` | **MUTATION** | Loads the queued markdown, invokes `revise()` or `contract()` on the AGM operator, appends a SUPERSEDE / REFINE event to the ledger, archives the markdown. **This is the one Ripple-adjacent path that mutates the graph — and it requires a `proposal_id` that was already queued + a `decision` from a human (or rule).** |
| `resolve_one(...)` | **MUTATION** | Bulk-mode entrypoint into `resolve_adjudication`. Same guarantees. |
| `find_pending_entry(...)` | **READ-ONLY** | Filesystem read of the queue dir. |
| `parse_decision(...)` / `parse_proposal_id(...)` | **READ-ONLY** | Pure string parsing. |

### `atlas_core/revision/` — the AGM operators

| Method | Class | What it does |
|---|---|---|
| `revise(driver, kref, new_content, ...)` | **MUTATION** | Creates a new `:AtlasRevision`, supersedes prior, retargets the `:AtlasTag` to the new revision. Implements AGM K\*2-K\*6. **Mutates the typed graph and the AGM closure.** |
| `contract(driver, kref, ...)` | **MUTATION** | Marks the current revision deprecated, removes from belief closure, preserves history per AGM Recovery rejection. **Mutates.** |
| `expand(driver, kref, content, ...)` | **MUTATION** | Adds a belief to the closure without superseding. **Mutates.** |
| `select_targets(...)` (`selection.py`) | **READ-ONLY** | Hansson Relevance/Core-Retainment selection. Reads candidates, returns chosen targets. Zero writes. |
| `Kref.parse(...)` (`uri.py`) | **READ-ONLY** | Pure URI parsing. |

### `atlas_core/trust/` — quarantine + ledger

| Method | Class | What it does |
|---|---|---|
| `QuarantineStore.upsert_candidate(claim, ...)` | **MUTATION (non-graph)** | Writes / merges a candidate row in the quarantine SQLite DB. Computes trust score, may auto-promote to `auto_promoted` status. **Does not touch the typed graph** — promotion to graph happens later via `promote_candidate` chained through `revise`. |
| `QuarantineStore.promote_candidate(candidate_id, ...)` | **MUTATION (non-graph)** | Marks the candidate as promoted in SQLite after `ledger.append_event`. Graph projection is a separate, retryable materializer step. |
| `materialize_approved_candidates(driver, quarantine)` | **MUTATION** | Idempotently projects every ledger-approved candidate as `(:Belief)-[:ABOUT]->(:AtlasItem)` in Neo4j. Ledger approval remains canonical; failed projections are reported and safe to retry. |
| `QuarantineStore.deny_candidate(...)` | **MUTATION (non-graph)** | SQLite status update only. |
| `QuarantineStore.upsert_dead_letter(...)` | **MUTATION (non-graph)** | SQLite write to the dead-letter table. |
| `QuarantineStore.list_pending(...)` / `list_requires_approval(...)` / `get_candidate(...)` | **READ-ONLY** | SQLite reads. |
| `HashChainedLedger.append_event(...)` | **MUTATION (non-graph)** | Appends a SHA-256-chained event to the ledger SQLite. **Does not touch the typed graph.** |
| `HashChainedLedger.verify_chain()` | **READ-ONLY** | Walks the chain top to bottom, verifies hash linkage, returns `ChainVerificationResult`. |
| `HashChainedLedger.is_promoted(...)` / `get_event(...)` / `get_root_lineage(...)` / `get_typed_root_state(...)` / `chain_length()` / `latest_event()` | **READ-ONLY** | SQLite reads. |

### `atlas_core/api/mcp_server.py` — the public MCP tool surface

The 17 tools registered in `_register_atlas_tools()`, classified:

| MCP tool name | Class | Notes |
|---|---|---|
| `ripple.analyze_impact` | **READ-ONLY** | Wraps `analyze_impact()`. |
| `ripple.reassess` | **PROPOSAL** | Wraps `reassess_cascade()`. Returns proposals; does not mutate. |
| `ripple.detect_contradictions` | **READ-ONLY** | Wraps `detect_contradictions()`. |
| `adjudication.queue` | **READ-ONLY** | Lists queued markdown entries. Filesystem read. |
| `adjudication.resolve` | **MUTATION** | The *one* MCP tool that mutates the graph. Calls `resolve_adjudication()` → `revise()` / `contract()` + ledger append. **Requires a `proposal_id` that already exists in the queue + an explicit `decision` arg.** |
| `quarantine.upsert` | **MUTATION (non-graph)** | Wraps `QuarantineStore.upsert_candidate()`. SQLite write only. |
| `quarantine.list_pending` | **READ-ONLY** | SQLite read. |
| `memory.search` | **READ-ONLY** | Deterministic lexical retrieval over non-denied SQLite candidates. |
| `memory.get` | **READ-ONLY** | Fetches one non-denied SQLite candidate. |
| `memory.list` | **READ-ONLY** | Lists non-denied SQLite candidates. |
| `memory.forget` | **MUTATION (non-graph)** | Marks the candidate denied so retrieval excludes it; preserves the audit row. |
| `ledger.verify_chain` | **READ-ONLY** | SQLite read + hash check. |
| `working_memory.assemble` | **READ-ONLY** | Loads working-memory blocks; no writes. |
| `lineage.walk` | **READ-ONLY** | Cypher MATCH traversal. |
| `sharing.grant` | **MUTATION (non-graph)** | Tier 5 multi-tenant policy write to SQLite. |
| `sharing.revoke` | **MUTATION (non-graph)** | Same. |
| `sharing.list_grants` | **READ-ONLY** | SQLite read. |

## What this means in practice

**For agent-runtime integrators (Hermes, OpenClaw, Claude Code):**
You can wire `ripple.analyze_impact`, `ripple.reassess`,
`ripple.detect_contradictions`, `adjudication.queue`,
`quarantine.list_pending`, `memory.search`, `memory.get`, `memory.list`,
`ledger.verify_chain`, `working_memory.assemble`, `lineage.walk`, and
`sharing.list_grants` as *read-only tools*. Your
agent can call them as often as it wants in a single turn; nothing in
Atlas's state changes. Only `adjudication.resolve`, `quarantine.upsert`,
`memory.forget`, and the two `sharing.{grant,revoke}` calls require
user-confirm gating.

**For Ripple-cascade callers:**
`RippleEngine.propagate()` is safe to invoke speculatively. It returns
proposals + queue entries. The graph is unchanged until a human (or a
rule you've configured) calls `adjudication.resolve` for some queued
proposal. Cascading in dry-run mode is just `propagate()` + reading
the result.

**For someone proving a soundness property:**
The set of methods that can change the typed graph is exactly
`{ revise, contract, expand, resolve_adjudication, resolve_one,
  adjudication.resolve (MCP wrapper) }`. AGM postulate compliance
follows from those operators alone, which is why they each carry a
property-based test in `tests/unit/test_agm_property_based.py` and an
end-to-end integration test in `tests/integration/test_agm_revise.py`.
The 49-scenario compliance suite (`docs/AGM_COMPLIANCE.md`) exercises
every postulate against `revise` + `contract` + `expand`.

**Naming convention forward:**
New methods that *propose* should follow the existing precedent —
`route_*`, `reassess_*`, `analyze_*`, `detect_*`. New methods that
*mutate* the graph should go through one of the three AGM operators
or be added to this audit document with a one-line justification of
why a new mutation surface was needed. New MCP tools that mutate
must include a `mutation: true` flag in their docstring and route
through `adjudication.resolve` when a human-in-the-loop decision
applies.
